## 🥅 Goals

1. 개발자로 성장하는데 도움이 될 고민들 해보기
2. 본인의 생각을 논리적으로 정리하는 연습
3. 공식적이고 신뢰있는 글을 쓰는 연습
4. 타인의 글을 읽고 질문하는 습관 만들기

## 🚀 Titles

1. 생각과 목표를 정리하는 글
    - (신입) 어떠한 개발자가 될 것인가
    - (기존) 협업에 대한 나의 가치관
    - (기존) 이전 기수에서 작성하지 않은 회고 글
2. 이론을 정리하는 글을 써보자.
    - CS지식 정리 (CS교육 정리 등)
    - 개발 프레임워크 소개
    - 언어에 대한 깊은 탐구 내용
    - IT 기업의 기술 블로그를 읽고 정리하기
3. 실전에 적용해보자.
    - 프로젝트를 하면서 겪었던 고민과 해결 방법과 그 이유
    - 알고리즘 문제 풀이 글
4. 소소한 고민과 생각들 ( 개발자로서든 학생에 관해서든 )
    - 코테이토 11기 회고록
    - 프로젝트 후기
    - 협업에서 중요하다고 생각하는 것들

## 📄 과제 제출 가이드

1. clone 혹은 remote 명령어 등을 이용해 리모트 리포지토리와 연결합니다.
2. `[자신의 github 아이디]-주차`와 동일한 이름의 branch로 체크아웃하여 작업합니다.
3. 최초 작업 시 본인의 브랜치명과 동일한 이름의 디렉토리를 만듭니다. (위 프로젝트 구조 참고)
4. 작업 완료 이후, 리모트 리포지토리의 `[자신의 github 아이디]-주차`와 동일한 이름의 브랜치에 push 합니다.
    - 이 때 merge는 스터디장 ‘신유승’이 진행하겠습니다. 
5. 과제 제출 기한에 맞춰 main branch로 PR을 날립니다. 블로그 스터디 리포지토리의 메인 브랜치명은 `develop`입니다.
6. PR은 `[본인 이름] N주차 과제 제출합니다.`의 제목으로 작성해주시면 됩니다.
7. PR 이후 로컬에서 작업 시 반드시 `git branch -d 브랜치 이름` 명령어로 로컬 브랜치를 삭제한다.

구체적인 PR방법은 아래 링크를 통해 확인 부탁드립니다.
